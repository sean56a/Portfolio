Desktop — No Display Output

A no-signal desktop diagnosed through systematic component isolation, tracing the fault to a dead discrete GPU.

2024 ~2.5 hours Resolved
device-photo-placeholder.jpg

01 Overview

A custom-built desktop powered on — fans spun, RGB lit up, drives audibly started — but no image ever reached the monitor, on any input or cable.

02 Problem

No display output on boot despite the system appearing to power on normally. No POST beep was audible, and the issue occurred with multiple monitors and cables ruled out beforehand by the owner.

03 Diagnosis

Because power delivery appeared normal, the fault was narrowed to either the GPU, RAM, or motherboard. A methodical component-isolation approach was used: testing with the motherboard's onboard graphics output (bypassing the discrete GPU entirely) to determine whether the GPU itself was the point of failure.

04 Tools Used

  • Anti-static wrist strap
  • Screwdriver set
  • Known-good PSU (for swap testing)
  • Spare RAM stick
  • POST debug LED / speaker
  • Compressed air

05 Repair Process

  • Disconnected all non-essential peripherals and reseated the GPU in its PCIe slot.
  • Reseated RAM modules one at a time to rule out a memory-related no-POST condition.
  • Connected the monitor to the motherboard's onboard video output instead of the GPU — display appeared successfully with full POST.
  • Reinstalled the GPU in a different PCIe slot to rule out a slot-specific fault — issue persisted.
  • Tested the GPU in a separate known-good system — confirmed no output there either, isolating the fault to the GPU itself.
  • Removed the failed GPU and configured the system to run on onboard graphics as an interim solution.

06 Testing

Confirmed stable POST and full desktop boot using onboard graphics across 10 restart cycles. Cross-tested the GPU in a second system to eliminate any doubt about the motherboard being the cause.

07 Outcome

The desktop was restored to a usable state via onboard graphics, with the discrete GPU confirmed dead and flagged for replacement or warranty claim.

08 Lessons Learned

Swapping one variable at a time — and testing suspected-faulty parts in a separate known-good system — is the most reliable way to avoid mis-diagnosing a multi-component failure as a single-part issue.
